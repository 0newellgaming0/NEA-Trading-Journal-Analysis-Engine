SEC EDGAR Intelligence Module (Institutional Filings & Signals Engine)
1. Overview

This module is a lightweight institutional-grade SEC EDGAR ingestion and signal extraction pipeline designed to enrich a trading system with regulatory intelligence.

It connects directly to the U.S. Securities and Exchange Commission’s EDGAR infrastructure and performs:

Ticker → CIK resolution
Filings ingestion (raw JSON)
Persistence of raw + processed data
Basic institutional signal extraction
Journal-ready formatted intelligence output

The system is intended to be embedded inside a larger trading framework (journal systems, scanners, regime engines, or confluence dashboards).

It is not a full fundamental analysis engine. It is a filings event + structural risk signal extractor.

2. Data Source

The module interacts with:

SEC Company Ticker Mapping API
https://www.sec.gov/files/company_tickers.json
EDGAR Submission Feed (CIK-based)
https://data.sec.gov/submissions/CIK{CIK}.json

These endpoints provide structured filing metadata including:

10-K / 10-Q reports
8-K corporate events
S-1 / S-3 / 424B5 equity issuance filings

4. Core Design Philosophy
4.1 Event-Driven, Not Fundamental Modeling

The system does NOT attempt valuation modeling.

Instead it tracks:

Filing frequency shifts
Regulatory event clustering
Dilution structure presence
Earnings cycle timing

This aligns with institutional trading workflows where:

“The filing itself is a signal, not the content of the filing.”

5. Function Breakdown
5.1 get_cik_from_ticker(ticker)
Purpose

Maps a stock ticker to its SEC CIK identifier.

Process
Fetches SEC ticker database JSON
Iterates lookup table
Returns zero-padded 10-digit CIK
Output
"0000320193"  # Example: AAPL
Failure Modes
API downtime → returns None
Missing ticker → returns None
5.2 get_filings(cik)
Purpose

Retrieves full EDGAR submission history for a company.

Endpoint
https://data.sec.gov/submissions/CIK{cik}.json
Returns

Raw JSON containing:

recent filings
historical filings
metadata arrays
5.3 save_raw_data(ticker, cik, filings_json)
Purpose

Persists full SEC response for reproducibility.

Stored Format
{
  "ticker": "TSLA",
  "cik": "0001318605",
  "timestamp": "2026-06-17T12:00:00",
  "data": { ... full SEC JSON ... }
}
Design Intent
Auditability
Backtesting filings-based strategies
Offline analysis capability
5.4 analyze_filings(filings_json)
Purpose

Extracts structured institutional signals from EDGAR data.

Input Structure
filings_json["filings"]["recent"]

Key arrays:

form
filingDate
Extracted Signals
Signal	Meaning
last_10k	Most recent annual report
last_10q	Most recent quarterly report
recent_8k_count	Corporate event frequency
dilution_events	Equity issuance / capital expansion pressure
Dilution Event Detection Logic

Triggered by forms:

S-1 (IPO / registration)
S-3 (shelf offerings)
424B5 (prospectus supplement / ATM issuance)

These are interpreted as:

Structural share supply expansion risk

8-K Interpretation

8-K filings are counted as:

Corporate events
M&A updates
leadership changes
material announcements

High frequency =

Elevated event-driven volatility regime

5.5 save_signals(ticker, signals)
Purpose

Stores processed institutional signals for downstream systems.

Output file:
sec_filings/{ticker}/signals.json
5.6 analyze_ticker_sec(ticker)
MAIN PIPELINE FUNCTION

This is the orchestration layer.

Execution Flow:
Resolve ticker → CIK
Fetch EDGAR filings
Save raw data snapshot
Run signal extraction
Save signals
Return structured object
Output Format
{
  "ticker": "AAPL",
  "cik": "0000320193",
  "signals": {
    "last_10k": "2025-11-02",
    "last_10q": "2026-02-01",
    "recent_8k_count": 4,
    "dilution_events": 1,
    "timestamp": "2026-06-17T12:00:00",
    "source": "sec_edgar"
  }
}
Failure Handling

Returns None if:

CIK not found
SEC API fails
No filings returned
Unexpected exception occurs

All failures are logged safely via log().

6. Institutional Interpretation Layer

The system encodes a simplified behavioral thesis:

6.1 Event Volatility Regime

High 8-K frequency implies:

News-driven price action
Reduced technical reliability
Wider intraday ranges
6.2 Dilution Pressure Regime

S-1 / S-3 / 424B5 implies:

Increased float supply
Potential sell pressure expansion
Long-term valuation compression risk
6.3 Earnings Cycle Positioning

10-K / 10-Q timestamps indicate:

Fundamental update timing
Earnings proximity
Narrative reset points
