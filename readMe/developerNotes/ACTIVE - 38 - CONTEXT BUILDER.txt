📌 Institutional Context Builder Module — README
Overview

The Institutional Context Builder is a presentation-safe orchestration layer that sits directly on top of the signalEngine module.

It is responsible for:

Converting raw analytical outputs into a clean, UI-safe, renderer-ready institutional context contract

This module does NOT perform analysis.

Instead, it acts as a:

Signal aggregator
Formatting safety layer
Contract enforcement system
Downstream compatibility bridge
🧠 Core Philosophy

This module is built around three strict design rules:

1. Analysis is External

All intelligence comes from:

signalEngine

This module does NOT compute:

trend logic
liquidity logic
volume logic
fib logic

It only orchestrates results

2. UI Must Never Receive Raw Dictionaries

A critical design constraint:

❌ No raw dicts in renderer layer
❌ No nested analysis objects in UI layer
✅ Only formatted primitives (strings, ints)

3. Context Pack is a Contract, Not a Suggestion

The returned object is a:

strict schema contract between engine → UI → journal system

Any deviation breaks downstream rendering consistency.

📦 Inputs
Main Function
build_institutional_context(df, f=float, volume_funcs=None)
Parameters
Parameter	Type	Description
df	DataFrame	OHLCV market data
f	callable	Numeric coercion function (default float)
volume_funcs	tuple	Optional plug-in volume system
Volume Plugin Contract

If provided:

(rvol, detect_volume_spike, institutional_state)

Where:

rvol → relative volume function
detect_volume_spike → spike detector
institutional_state → institutional classification function
🧠 Processing Pipeline

The module executes a fixed deterministic pipeline:

1. Trend Evaluation
evaluate_trend(df, f)

Outputs:

trend direction (Bullish / Bearish / Neutral)
strength score
2. Liquidity Sweep Detection
detect_liquidity_sweep(df)

Detects:

Spring (bear trap)
Upthrust (bull trap)
3. Structure Evaluation
evaluate_structure(df)

Detects:

Near Support
Near Resistance
Neutral
4. Fibonacci Alignment
evaluate_fibonacci(df)

Detects:

price alignment zones (38.2 / 50 / 61.8 / 78.6)
5. Volume Confirmation Layer

Two execution modes:

A. Default Mode (No Volume Plugin)

If volume_funcs is None:

{
    "confirmed": False,
    "score": 0,
    "rvol": 0.0,
    "institutional_state": "UNKNOWN"
}
B. Active Volume Mode

If plugin provided:

volume_confirmation(
    df,
    rvol,
    detect_volume_spike,
    inst_state
)
