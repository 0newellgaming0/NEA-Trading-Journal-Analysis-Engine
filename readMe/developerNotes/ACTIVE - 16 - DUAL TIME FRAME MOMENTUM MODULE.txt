1. OVERVIEW

The Dual Time Frame Momentum Module (DTFM) is an institutional-grade analytics engine designed to evaluate momentum, structure, volatility, and volume behavior across two key timeframes:

60-Minute (Intraday Structure)
Daily (Macro Structure)

Its primary objective is to detect:

Momentum alignment or divergence across timeframes
Expansion vs contraction of volatility and participation
Structural positioning within recent price ranges
Early signals of trend continuation, exhaustion, or distribution risk

The module outputs a fully formatted journal-ready analytical report suitable for trading dashboards, systematic workflows, or discretionary decision support systems.

2. CORE CONCEPT

The engine operates under a simple institutional principle:

“True trend quality is only confirmed when lower timeframe momentum aligns with higher timeframe structure and participation.”

It does not rely on indicators alone. Instead, it evaluates:

Price momentum (returns + slope)
Volatility expansion (range behavior)
Volume expansion (participation shifts)
Structural positioning (where price sits within range)
Cross-timeframe regime alignment

7. REGIME CLASSIFICATION

The engine assigns one of four structural regimes:

7.1 BULLISH ALIGNMENT
60M > 0 AND Daily > 0
Indicates synchronized upward momentum
7.2 BEARISH ALIGNMENT
60M < 0 AND Daily < 0
Indicates synchronized downward pressure
7.3 COUNTERTREND RECOVERY
60M > 0 AND Daily < 0
Intraday bounce against macro trend
7.4 DISTRIBUTION RISK
60M ≤ 0 AND Daily ≥ 0
Potential exhaustion or hidden selling pressure
8. INTERPRETATION ENGINE

The system generates qualitative insights based on thresholds:

Momentum Signals
60M outperforming Daily → short-term strength
Participation Signals
Volume ratio > 1.20 → institutional involvement increasing
Volatility Signals
Range ratio > 1.20 → expansion phase detected
Structural Signals
Position > 80% → price near upper boundary
Position < 20% → price near lower boundary

These are compiled into a journal-ready explanation block.

10. ERROR HANDLING

The module uses defensive returns instead of exceptions.

Error Cases:
Missing Column
Identifies exact missing column
Specifies timeframe (60M or Daily)
Insufficient Data
Requires ≥ 30 rows per timeframe
Runtime Exception
Captures and returns exception message safely

All errors are returned as structured text blocks for dashboard compatibility.
