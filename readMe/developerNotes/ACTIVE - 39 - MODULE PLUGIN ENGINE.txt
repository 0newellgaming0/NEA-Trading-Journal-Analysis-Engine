📊 Candlestick State Engine
Overview

The Candlestick Institutional State Engine is a modular multi-pattern candlestick analysis framework designed to detect, normalize, and structure institutional-grade price action signals across multiple candlestick systems.

It integrates multiple independent pattern detectors into a unified execution pipeline that produces:

Structured journal-ready prompts
Standardized analysis outputs
Time-stamped historical records
CSV exports organized by ticker and time partition

The system is designed for multi-timeframe trading workflows, especially those focused on:

Wyckoff-style interpretation
Liquidity-based price action
Institutional candlestick behavior
Reversal and continuation pattern recognition
🧠 Core Design Philosophy

This engine follows a 3-layer architecture:

1. Raw Module Layer

Each candlestick module runs independently:

Pinbar
Hammer
Engulfing
Inside Bar
Marubozu
Star patterns
Harami
Three Inside patterns
DCCPL reversal systems

Each module returns:

Either a dictionary
A string
Or a raw object
2. Normalization Layer

The system standardizes all outputs into:

{
    "ticker": str,
    "module": str,
    "journal_prompt": str,
    "timestamp": ISO_UTC
}

This ensures:

Cross-module compatibility
UI consistency
Database-ready structure
CSV export integrity
3. Persistence Layer

Outputs are saved into structured CSV files:

/data/candlestickAnalysis/<TICKER>/<YYYY-MM>/

Each file contains:

Full multi-module analysis snapshot
Timestamped execution record
Journal-ready interpretation prompts
📦 Module Registry

The engine dynamically executes all registered candlestick modules:

Module Name	Function
Pinbar	analyze_pinbar

Marubozu	analyze_marubozu

DCCPL	analyze_reversal_patterns

Each module is executed independently and contributes to the final dataset.
