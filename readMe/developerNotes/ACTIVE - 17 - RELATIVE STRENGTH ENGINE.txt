RELATIVE STRENGTH ENGINE
BLOCK-BASED INSTITUTIONAL RELATIVE PERFORMANCE MODULE
=====================================================

AUTHOR:
Newell Trading Group

MODULE PURPOSE:
relativeStrength.py

This module provides a benchmark-relative performance analysis engine designed to measure how a stock is performing against major market indices and benchmark baskets.

Unlike traditional price analysis modules that evaluate a stock in isolation, the Relative Strength Engine evaluates:

```
"How well is this asset performing relative to the broader market?"
```

The engine is designed to support:

• Institutional Rotation Analysis
• Relative Performance Measurement
• Leadership Detection
• Benchmark Sensitivity Mapping
• Capital Flow Analysis
• Risk-On / Risk-Off Participation Assessment
• Sector and Index Rotation Monitoring
• Journal Prompt Generation
• Confluence Dashboard Integration

The module outputs a structured institutional-style narrative that can be directly inserted into:

• Journal Analysis
• Trade Reports
• Dashboard Panels
• Institutional Research Notes
• Confluence Engines
• AI Prompt Builders

---

## CORE CONCEPT

Price alone does not reveal whether a stock is outperforming or underperforming the market.

Example:

Stock A:
+20%

SPY:
+30%

Although Stock A gained 20%, it actually underperformed the market.

Institutional traders continuously compare assets against benchmark baskets to determine:

• Capital efficiency
• Opportunity cost
• Relative leadership
• Rotation probability

This module automates that process.

1. BENCHMARK DEFINITIONS

---

BENCHMARK_GROUPS

Defines benchmark baskets used for relative comparisons.

Current configuration:

MEGA CAP

```
SPY
QQQ
DIA
```

GROWTH

```
QQQ
VUG
```

BROAD MARKET

```
SPY
VTI
```

SMALL CAP

```
IWM
IJR
```

Example:

BENCHMARK_GROUPS = {
"mega_cap": ["SPY", "QQQ", "DIA"],
"growth": ["QQQ", "VUG"],
"broad_market": ["SPY", "VTI"],
"small_cap": ["IWM", "IJR"]
}

Purpose:

Allows a stock to be compared across multiple liquidity regimes rather than a single benchmark.

Institutional advantage:

A stock can outperform growth benchmarks while underperforming small-cap benchmarks.

This reveals rotational behavior that a single benchmark cannot detect.

6. RELATIVE STRENGTH CALCULATION ENGINE

---

Function:

compute_rs()

Purpose:

Calculates benchmark-relative performance.

Formula:

RS = Asset Price / Benchmark Price

Example:

AAPL = 220

SPY = 550

RS = 220 / 550

Interpretation:

Higher values indicate stronger relative performance.

---

## TREND CALCULATION

The engine measures RS momentum.

Formula:

Current RS - RS 10 Bars Ago

Example:

Current RS = 0.450

Prior RS = 0.400

Trend = +0.050

Interpretation:

Positive:
Relative strength improving

Negative:
Relative strength deteriorating

---

## MINIMUM DATA REQUIREMENT

The engine requires:

20+ bars

Reason:

Short datasets produce unstable RS readings.

If fewer than 20 observations exist:

Returns None

## LEADERSHIP DETECTION

Strongest Benchmark

The benchmark against which the stock exhibits the highest relative performance.

Used for:

• Leadership mapping
• Rotation analysis
• Institutional concentration analysis

---

## WEAKEST BENCHMARK

Identifies where performance is weakest.

Useful for:

• Detecting structural drag
• Locating rotation outflows
• Identifying underperforming market regimes

---

## ROTATION ANALYSIS

Improving Count

Benchmarks with positive RS trend

Weakening Count

Benchmarks with negative RS trend

Example:

Improving = 6

Weakening = 2

Interpretation:

Positive rotational bias

Institutional implication:

Capital may be rotating into the asset.

---

## FLOW STATE CLASSIFICATION

Positive Rotation Bias

Improving > Weakening

Negative Rotation Bias

Weakening > Improving

Balanced Rotation

Improving == Weakening

Used to evaluate:

• Capital migration
• Sector rotation
• Relative momentum shifts

---

## REGIME CLASSIFICATION

Leadership Regime

avg_rs > 1.05

Interpretation:

Asset is outperforming benchmark basket.

Neutral Regime

0.98 < avg_rs <= 1.05

Interpretation:

Tracking market performance.

Lagging Regime

avg_rs <= 0.98

Interpretation:

Underperforming benchmark basket.
