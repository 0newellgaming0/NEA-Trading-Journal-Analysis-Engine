Module: Stop Loss Evaluation & T-Line Risk Engine
Version: 1.0 (Institutional Risk Layer – Multi-Timeframe T-Line Integration)
1. Overview

This module implements a multi-layered institutional stop-loss evaluation engine designed for systematic trading workflows using:

Structural stop levels (manual / strategy-defined risk)
T-Line (EMA-8 based trend stop logic)
Multi-timeframe confirmation (60-minute + Daily)
Regime classification:
VALID
PULLBACK
BREACHED

It is intended to function as a final decision gate inside a trading journal, risk engine, or execution system, ensuring trades are continuously validated against both structural and trend-based risk frameworks.

2. Core Philosophy

This module is built on a hybrid of:

Wyckoff-style structural risk control
T-Line (EMA-8) momentum integrity
Multi-timeframe confirmation hierarchy

Instead of a single stop-loss, the system evaluates:

“Is the trade structurally invalid, temporarily pulling back, or fully broken across timeframes?”

This avoids premature exits during healthy pullbacks while still enforcing strict risk discipline during true breakdowns.

5.4 evaluate_stop_loss(latest_close, stop_val, df, daily_df, ticker)
🔴 CORE ENGINE FUNCTION

This is the primary risk decision engine.

Inputs
Parameter	Type	Description
latest_close	float	Current market price
stop_val	float	Structural stop level
df	DataFrame	60-minute timeframe data
daily_df	DataFrame	Daily timeframe data
ticker	str	Symbol identifier
Multi-Layer Evaluation System
Layer 1 — Structural Stop
structural_breached = latest_close < stop_val

This represents:

Hard invalidation level
Strategy-defined risk point
Layer 2 — 60M T-Line Stop
tline_stop = get_tline_stop(df)
tline_breached = latest_close < tline_stop

Represents:

Intraday trend breakdown
Momentum failure on lower timeframe
Layer 3 — Daily T-Line Stop
daily_tline_stop = get_tline_stop(daily_df)
daily_tline_breached = latest_close < daily_tline_stop

Represents:

Higher timeframe structural breakdown
Institutional trend failure
State Classification Logic

The engine assigns one of three states:

🟢 VALID

Conditions:

No structural break
No higher timeframe breakdown

Interpretation:

Trend intact
No risk trigger activated
🟡 PULLBACK

Condition:

tline_breached == True
AND daily_tline_breached == False

Interpretation:

Intraday weakness
Higher timeframe structure intact
Controlled retracement, NOT failure

This is critical for:

Avoiding premature exits
Holding during normal volatility
🚨 BREACHED

Condition:

structural_breached == True
OR daily_tline_breached == True

Interpretation:

Full structural breakdown
Institutional exit signal
Risk containment required immediately
Output Structure
VALID / PULLBACK / BREACHED all return:
{
    "breached": bool,
    "status": str,
    "state": str,
    "structural_breached": bool,
    "tline_breached": bool,
    "daily_tline_breached": bool,
    "tline_stop": float | None,
    "daily_tline_stop": float | None,
    "block": str
}

Preserve decision integrity over computation completeness

T-Line → skipped
State → downgraded to safe evaluation
Risk system remains operational
