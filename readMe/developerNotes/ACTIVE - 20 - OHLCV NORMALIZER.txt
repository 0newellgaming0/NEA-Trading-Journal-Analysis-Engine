Module: Unified Timestamp + Dynamic OHLCV Normalizer
File: ohlcv_normalizer.py (or equivalent)
1. Overview

This module provides a robust, institution-grade preprocessing layer for OHLCV market data. Its primary responsibility is to:

Normalize inconsistent timestamp formats into a unified UTC datetime index
Dynamically detect OHLCV columns without relying on strict naming conventions
Standardize market data into a clean schema:
High, Low, Close, Volume
Enforce strict data validation rules to prevent silent corruption in downstream trading engines

It is designed to act as a mandatory preprocessing gate before any Wyckoff, momentum, liquidity phase, or institutional flow analysis.

2. Design Philosophy

This module follows three core principles:

2.1 No Assumptions

Market data sources are inconsistent. This module avoids hardcoded column names like:

open
high
close
volume

Instead, it uses pattern-based detection.

2.2 Fail Loud, Not Silent

Unlike typical pandas pipelines, this module:

Raises explicit exceptions on invalid data
Rejects NaN contamination immediately
Rejects insufficient history instead of padding or guessing

This ensures downstream signals (Wyckoff phases, liquidity shifts, etc.) remain institutionally reliable.

2.3 Canonical Output Schema

All outputs are forced into a standardized format:

Column	Type	Description
High	float	Bar high price
Low	float	Bar low price
Close	float	Closing price
Volume	float	Trading volume
timestamp	datetime (UTC)	Sorted time index
3. Function: normalize_timestamp(df)
3.1 Signature
def normalize_timestamp(df: pd.DataFrame) -> pd.DataFrame
4. Input Requirements
4.1 Input Type
Must be a pandas.DataFrame
4.2 Required Time Column (one of):

The function automatically detects one of:

timestamp
datetime
date

All are converted into:

UTC datetime (pandas datetime64[ns, UTC])
4.3 OHLCV Columns

The function dynamically searches for columns matching:

Matching Logic:

A column is valid if it:

starts with "high", "low", "close", "volume"
OR ends with "high", "low", "close", "volume"
OR exactly equals the prefix (case-insensitive)
Examples Supported:
high
high_rbbn
rbbn_high
closeTSLA
volume_x
x_volume
5. Processing Pipeline
Step 1 — Validation
Reject if:
- df is None
- df is not a DataFrame
Step 2 — Timestamp Normalization
Detect time column
Convert using:
pd.to_datetime(..., errors="coerce", utc=True)
Drop invalid timestamps
Sort ascending
Reset index
Step 3 — Dynamic Column Detection

Finds OHLCV columns using flexible pattern matching.

If any are missing:

Missing Field	Error Raised
High	Missing High column
Low	Missing Low column
Close	Missing Close column
Volume	Missing Volume column
Step 4 — Standardization

Creates canonical columns:

df["High"]
df["Low"]
df["Close"]
df["Volume"]

All values are converted using:

pd.to_numeric(errors="coerce")
Step 5 — Hard Validation

The system enforces strict integrity checks:

1. NaN Check
if df[required].isna().any().any()

If any NaNs exist → Exception raised

2. Minimum History Check
if len(df) < 30:

Rejects datasets with fewer than 30 bars.

This ensures:

Wyckoff phase detection stability
Moving average validity
Volume regime reliability
6. Output

Returns a cleaned and standardized DataFrame:

return df
Guaranteed Structure:
Sorted by timestamp (ascending)
UTC-normalized time index
Clean OHLCV numeric fields
No NaN values
Minimum 30 bars of history
