📘 Institutional Reversal Engine — README
Overview

This module implements a candlestick-based institutional reversal detection system focused on two classic price action patterns:

🌩 Dark Cloud Cover (bearish reversal)
☀️ Piercing Line (bullish reversal)

It is designed for institutional-style signal detection, combining:

OHLC normalization
Pattern recognition
State memory tracking
Multi-day event confirmation logic
Trade plan generation
Journal-ready output formatting

The system is built for integration into trading dashboards, quant research pipelines, and journal automation tools.

🧠 Core Design Philosophy

This module treats price action as:

An auction of liquidity transitioning between institutional control zones

Rather than simple candlestick detection, it introduces:

Stateful reversal tracking
Confirmation / failure logic
Time-decayed signal validity
Trade planning based on structure, not emotion
📦 Module Structure
1. Safe Scalar Extraction (f)
Purpose

Ensures robust extraction of numeric values from:

pandas Series
scalars
NaN values

f(x) -> float
Logic

Why it matters

Broken comparisons in candle detection

🌩 4. Dark Cloud Cover Detection
Function
detect_dark_cloud_cover(df)
Pattern Logic

A bearish reversal pattern requiring:

Conditions:
Previous candle is bullish
Current candle is bearish
Gap up opening above prior high
Close penetrates below midpoint of previous candle
Close remains above previous open
Midpoint Formula
midpoint = prev_open + (prev_close - prev_open) * 0.5
Output Example
{
  "detected": true,
  "type": "DarkCloudCover",
  "strength": "Institutional Reversal",
  "high": 10.4,
  "low": 9.8
}
☀️ 5. Piercing Line Detection
Function
detect_piercing_line(df)
Pattern Logic

A bullish reversal pattern requiring:

Conditions:
Previous candle is bearish
Current candle is bullish
Gap down below prior low
Close penetrates above midpoint
Close remains below previous open
Output mirrors Dark Cloud Cover structure:

}

⚙️ 7. Event Engine (State Machine)
Function
build_reversal_event_state(df, max_confirm_days=5)
Purpose

Transforms raw signals into multi-day institutional events

Event Lifecycle
1. PENDING

Triggered when pattern appears

2. CONFIRMATION RULES
Dark Cloud Cover:
CONFIRMED → price breaks BELOW low
FAILED → price breaks ABOVE high
Piercing Line:
CONFIRMED → price breaks ABOVE high
FAILED → price breaks BELOW low
3. EXPIRATION RULE

If no resolution within:

max_confirm_days = 5

Event → EXPIRED

Output
events = [
  {
    "type": "DarkCloudCover",
    "status": "CONFIRMED",
    "start_index": 10,
    "days_active": 3
  }
]
📊 8. Trade Plan Engine
Function
build_reversal_trade_plan(today, yesterday, state)
Purpose

Generates structured trade setups based on:

Pattern type
Confirmation state
Price range (high-low structure)
Trade Logic
🟡 Pending Event
Entry = breakout boundary
Stop = opposite extreme
No targets yet (uncertain state)
🌩 Dark Cloud Cover (bearish)
Entry = low of signal candle
Stop = high of signal candle
Targets = downward extensions
☀️ Piercing Line (bullish)
Entry = high of signal candle
Stop = low of signal candle
Targets = upward extensions
Example Output

  "setup": "FORMING PIERCING LINE",
  "entry": 10.50,
  "stop_close": 9.80,
  "target1": 11.20,
  "target2": 11.90
}
