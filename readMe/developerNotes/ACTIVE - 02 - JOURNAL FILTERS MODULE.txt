MODULE NAME

journal_filters.py

OVERVIEW

The journal_filters.py module provides a lightweight but powerful filtering engine for a trading journal UI system built around a Treeview-based interface (typically Tkinter).

Its primary responsibility is to:

Filter journal entries dynamically based on user input
Support column-specific searching
Render filtered results into a UI Treeview
Provide preview-safe truncation for large text fields (notes fields)
Update real-time result counts in the interface

This module is designed for interactive trading journal dashboards, where rapid filtering of trade history, analysis notes, and management logs is required.

CORE FUNCTION
apply_journal_filters(...)

This is the main and only function in the module.

It processes journal data and updates the UI Treeview based on user-selected filters and search input.

FUNCTION SIGNATURE
apply_journal_filters(
    tree,
    journal_rows,
    filter_search_var,
    filter_type_var,
    CSV_FIELDS,
    preview_text,
    filter_count_label
)
PARAMETERS
1. tree

Type: tkinter.ttk.Treeview

The UI component where filtered journal rows are displayed.

All previous rows are cleared before applying filters
New filtered rows are inserted dynamically
2. journal_rows

Type: list[dict]

A list of journal entries, where each entry is a dictionary:

{
    "ticker": "AAPL",
    "entry_price": 150.0,
    "trade_notes": "...",
    "analysis_notes": "...",
    ...
}

Each dictionary represents one trade record.

3. filter_search_var

Type: tkinter.StringVar

Holds the current search query entered by the user.

Automatically converted to lowercase
Whitespace is stripped before filtering
4. filter_type_var

Type: tkinter.StringVar

Defines which field/column is being searched.

Example values:

"ticker"
"trade_notes"
"strategy"
"analysis_notes"

This enables column-specific filtering.

5. CSV_FIELDS

Type: list[str]

Defines the column order and structure of displayed data.

Example:

CSV_FIELDS = [
    "ticker",
    "entry_price",
    "exit_price",
    "trade_notes",
    "analysis_notes"
]

This ensures Treeview rows are consistently formatted.

6. preview_text

Type: callable

A helper function used to truncate long text fields.

Primarily applied to:

trade_notes
analysis_notes
management_notes
Purpose:

Prevents UI clutter by shortening large text blocks.

Example behavior:

Original: "This trade shows institutional accumulation with breakout confirmation..."
Preview:  "This trade shows institutional..."
7. filter_count_label

Type: tkinter.Label

Displays the number of visible filtered results.

Example output:

12 Result(s)
HOW IT WORKS (LOGIC FLOW)
STEP 1 — CLEAR UI TREE
tree.delete(*tree.get_children())

Removes all previously displayed rows to prepare for fresh filtered output.

STEP 2 — READ FILTER INPUTS
search_text = filter_search_var.get().strip().lower()
filter_type = filter_type_var.get()
Normalizes search text
Retrieves selected column filter
STEP 3 — INITIALIZE COUNTER
visible_count = 0

Tracks how many rows match the filter conditions.

STEP 4 — LOOP THROUGH JOURNAL ROWS
for row in journal_rows:

Each journal entry is evaluated individually.

STEP 5 — COLUMN-BASED MATCHING
compare_value = str(row.get(filter_type, "")).lower()
Extracts value from selected column
Converts to lowercase
Ensures safe string comparison
STEP 6 — FILTER CONDITION
if search_text in compare_value:

A partial match system is used:

✔ Supports substring search
✔ Case-insensitive matching
✔ Works across all selected fields

STEP 7 — FORMAT ROW FOR DISPLAY
visible_values = []

For each field in CSV_FIELDS:

Extract value from row
Apply preview truncation if needed
Append to display list
Special Handling:
if field in ("trade_notes", "analysis_notes", "management_notes"):
    value = preview_text(value)

This ensures large text does not break UI layout.

STEP 8 — INSERT INTO TREEVIEW
tree.insert("", "end", values=visible_values)

Adds the filtered row into the UI display.

STEP 9 — UPDATE RESULT COUNT
visible_count += 1

Tracks number of matching entries.

STEP 10 — UPDATE UI LABEL
filter_count_label.config(
    text=f"{visible_count} Result(s)"
)

Provides real-time feedback to the user.

KEY FEATURES
1. COLUMN-SPECIFIC SEARCHING

Users can filter by specific fields instead of searching globally.

2. PARTIAL MATCH FILTERING

Search does NOT require exact matches:

"tesla" matches "tesla inc"
"breakout" matches "strong breakout confirmed"
3. UI SAFE TEXT HANDLING

Long notes fields are automatically shortened.

4. REAL-TIME RESULT COUNT

Users always know how many entries match their filter.

5. SIMPLE IN-MEMORY OPERATION

No database dependency — operates entirely on:

Python lists
dictionaries
Tkinter UI components
PERFORMANCE CHARACTERISTICS
Time Complexity: O(n × m)
n = number of journal rows
m = number of CSV fields
Optimized for:
Small to medium trading journals
Real-time UI filtering
Desktop dashboards
DEPENDENCIES

This module assumes integration with:

Tkinter (Treeview, Label, StringVar)
Journal data loaded as list of dictionaries
A text preview helper function
