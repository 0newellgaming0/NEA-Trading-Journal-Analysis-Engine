Stock Data Downloader + Financial Ingestion System
README.txt
🧠 OVERVIEW

This module is a multi-timeframe market data ingestion engine + financial statement pipeline + Tkinter dashboard UI built on top of:

yfinance (market data + fundamentals)
SQLite (primary structured storage layer)
CSV (backup layer)
Tkinter (manual control dashboard)
Multi-threaded update engine

It is designed as a core data backbone for a trading system, feeding:

OHLCV datasets (multi-timeframe)
Institutional-grade financial statements
Watchlist-driven ticker updates
Historical data repair + append logic
🏗️ ARCHITECTURE
🔹 Data Flow Pipeline
watchlist.db (source of tickers)
        ↓
yfinance API (market + fundamentals)
        ↓
append/merge engine (OHLCV repair logic)
        ↓
StockDataRepository (SQLite PRIMARY storage)
        ↓
CSV backup files (secondary persistence)
        ↓
financials/ directory (fundamental JSON + DB storage)

📊 WATCHLIST SYSTEM
Source of truth:
watchlist.db → table: watchlist
Query:
SELECT ticker FROM watchlist
Behavior:
Deduplicates tickers
Normalizes to uppercase
Sorted before processing
📥 MARKET DATA INGESTION ENGINE
Function:
append_new_data(ticker, tf_name, cfg)
Features:
1. Live Data Pull

Uses:

yf.download()
2. Intelligent Merge System

Prevents data duplication using:

last timestamp comparison
overlap correction window (50 bars)
incremental delta insertion
duplicate-safe merge
3. Repair Logic

If overlap is detected:

splits old dataset into stable + refresh zones
merges new incremental data
removes duplicates by timestamp
4. Safety Trim
if len(df) > 1500:
    keep last 1500 rows

Prevents database bloat.

🧱 STORAGE LAYERS
1. PRIMARY STORAGE (SQLite)

Handled by:

StockDataRepository.insert_ohlcv_df()

Also logs ingestion:

repo.log_ingestion(ticker, tf_name, row_count, status)
2. SECONDARY STORAGE (CSV BACKUP)

Saved to:

/modules/stock_data/{timeframe}/{TICKER}.csv

Used for:

manual recovery

fallback persistence

SQLite financial tables
