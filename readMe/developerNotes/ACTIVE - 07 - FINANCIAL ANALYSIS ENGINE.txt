Financial Analysis Engine Module
Overview

This module provides a lightweight but structured fundamental financial analysis engine designed to operate on locally stored financial statement CSV files. It is built to integrate directly into a larger trading or journal system (such as journal.py) and produces both:

A structured analysis dictionary (analyze_financials)
A formatted human-readable prompt block (format_financial_prompt)

The system focuses on institutional-style financial quality assessment, combining income statement strength, balance sheet health, and cash flow stability into a simple 5-point scoring model.

It is intentionally designed to be:

Dependency-light (only os, pandas, datetime)
Fail-safe (graceful handling of missing or malformed data)
Integration-ready (returns structured outputs for dashboards and journal systems)

Each ticker folder must contain three standardized CSV files.

Data Format Expectations

Each CSV is expected to be structured in a row-based financial statement format, where:

Row index = financial metric name (string key)
Column 0 = latest reported value (primary value used)

Example:

Metric	Value
Total Revenue	1200000
Net Income	250000

The module always pulls the first column (iloc[0]) of each row.

Core Functions
1. load_csv(ticker, filename)

Loads a financial statement CSV for a given ticker.

Behavior:

Builds path dynamically from FINANCIALS_DIR
Returns pandas.DataFrame if file exists
Returns None if missing or unreadable

Failure handling:

Safe fallback (None) prevents system crashes
2. get_latest(df, row_name)

Extracts the most recent value for a specific financial metric.

Logic:

Locates row by index name
Returns first column value (iloc[0])
Returns None if:
DataFrame is missing
Row does not exist
Exception occurs

This function acts as a universal safe extractor for financial statements.

Institutional Quality Score (5-Point System)

The module computes a simplified fundamental strength score (0–5):

Scoring Rules:

+1 if:

Operating Cash Flow > 0
Net Income > 0
Revenue > 0
Net Margin > 5%
Debt-to-Equity < 2
Interpretation:
Score	Meaning
0–1	Weak fundamentals
2–3	Neutral / mixed quality
4	Strong company
5	Institutional-grade quality

This is designed as a heuristic filter, not a valuation model.

System Design Philosophy

This module follows three core design principles:

1. Fail-Safe Data Handling

Every function gracefully degrades rather than throwing errors.

2. Institutional Simplicity

Instead of complex valuation models, it focuses on:

Profitability
Cash flow strength
Leverage risk
3. Journal Integration First

Output is structured for:

Trading journals
Dashboard panels
Strategy engines
Backtesting systems
Integration Notes (journal.py)

This module is designed to plug directly into:

Trade journal entry generation
Ticker analysis pipelines
Institutional scoring dashboards

Typical usage:

from financials_engine import format_financial_prompt

print(format_financial_prompt("AAPL"))
Limitations
No real-time API fetching (CSV-based only)
Assumes correct financial statement formatting
Does not normalize currencies
No multi-period trend analysis (single snapshot only)

Summary

This module serves as a lightweight institutional fundamentals engine that transforms raw financial statement CSVs into:

Structured analytics
Risk-aware ratios
A simplified institutional quality score
Journal-ready narrative output

It is optimized for trading system integration rather than accounting precision, prioritizing clarity, speed, and usability inside decision engines.
