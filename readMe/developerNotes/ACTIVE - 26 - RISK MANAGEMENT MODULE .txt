==========================================================
RISK ENGINE MODULE (Tkinter-Based Position Sizing System)
==========================================================
MODULE OVERVIEW

This module implements a stateful trading risk engine built on top of tkinter.Variable objects. It is designed for integration into a GUI-based trading journal and execution system.

The engine handles:

Dynamic position sizing (risk-based allocation)
Multi-entry ladder pricing system
Automated stop-loss calculation (Wyckoff-style volatility model)
Risk-to-reward target generation
“Buy Now” execution sizing
Full state synchronization with external journal systems
Safe initialization guards for GUI lifecycle stability

It is designed as a reactive computation engine, meaning all key outputs automatically update when inputs change.

CORE DESIGN PRINCIPLES
1. Reactive State System

All variables are stored as tk.DoubleVar or tk.StringVar, enabling automatic UI binding.

2. Risk-First Architecture

Position sizing is always derived from:

risk_dollar = account * risk_pct

Everything downstream depends on this value.

3. Ladder-Based Entry System

The system supports 4 entry levels:

ladder_prices[0–3]
ladder_shares[0–3]
ladder_totals[0–3]

These are derived from Fibonacci-style extensions of the price range.

4. Institutional Stop Logic

Stop loss is not static—it is computed using:

range compression
shakeout buffer
volatility scaling
minimum loss constraints

STOP LOSS ENGINE
auto_calc_stop(*args)

This is the core institutional stop engine.

Inputs:
last_high
low
Step 1: Range calculation
price_range = last_high - low
Step 2: Shakeout model
shakeout_level = low - (price_range * 0.075238)

This simulates:

liquidity sweep behavior
Wyckoff-style spring logic
Step 3: Volatility-based buffer

Buffer scales dynamically:

Range % of High	Buffer
< 10%	6%
< 20%	9%
< 35%	12%
> 35%	18%
Step 4: Final stop
calc_stop = shakeout_level - stop_buffer
Step 5: Risk constraints
Must not exceed 65% drawdown from low
Must not go below zero
If invalid → fallback to low * 0.50
Output:
stop.set(round(calc_stop, 4))
MAIN CALCULATION ENGINE
recalc(*args)

This is the central financial engine.

It recalculates EVERYTHING in real time.

1. Risk Dollar Calculation
risk_dollar = account * risk_pct

This defines total capital at risk.

2. Ladder Price Construction

Uses Fibonacci-style extensions:

fib_levels = [0.328, 0.238, 0.015, -0.075238]

Each ladder price:

price = low + (range * fib_level)

Special case:

negative extension is bounded to prevent invalid pricing
3. Position Sizing Engine

Allocation model:

[0.25, 0.75, 0, 0]

Meaning:

1st ladder = 25% risk
2nd ladder = 75% risk
others unused by default
Share Calculation:
shares = (risk_allocated_dollars) / (entry_price - stop)

Ensures:

constant risk per trade
dynamic position scaling
4. Total Portfolio Metrics

Computed:

total_shares
total_cost
5. Risk-to-Reward Targets

For each ladder:

rr_target = entry + (entry - stop)

This enforces a 1:1 risk-reward baseline

6. Buy Now Engine
risk_per_share = buy_now_price - stop
shares = risk_dollar / risk_per_share

Outputs:

buy_now_shares
buy_now_total
