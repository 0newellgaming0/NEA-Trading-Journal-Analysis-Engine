# ACCEPTANCE_PHASE_ENGINE README

## MODULE NAME

acceptance_phase_engine.py

## PURPOSE

The Acceptance Phase Engine is an institutional market structure analysis module designed to identify periods of market equilibrium, value acceptance, and balanced auction behavior.

Unlike traditional trend-following systems that focus on directional movement, this engine focuses on determining whether price is currently being accepted by market participants.

The module is based on auction market theory concepts and institutional order-flow principles where markets alternate between:

1. Discovery Phases
2. Acceptance Phases
3. Expansion Phases
4. Distribution Phases
5. Exhaustion Phases

This engine specifically identifies ACCEPTANCE CONDITIONS.

Acceptance occurs when buyers and sellers reach temporary agreement on value, producing:

• Stable price behavior
• Reduced directional conviction
• Balanced participation
• Contained volatility
• Lack of breakout activity

The engine is designed for use inside:

• Trading Dashboards
• Market Regime Engines
• Institutional Analysis Systems
• Journal Systems
• Multi-Factor Confluence Models
• Trade Filtering Frameworks

=================================================================
CORE PHILOSOPHY
===============

Institutional traders view markets as auctions.

Price constantly searches for liquidity and fair value.

Markets generally operate in two broad states:

## DISCOVERY

Price is actively searching for new value.

Characteristics:

• Trend expansion
• Volatility expansion
• Range extension
• Breakouts
• Aggressive directional movement

## ACCEPTANCE

Price has found temporary agreement.

Characteristics:

• Balance
• Compression
• Stable participation
• Contained volatility
• Two-sided trade

This module attempts to determine when a market has shifted into the second state.

=================================================================
ANALYSIS PIPELINE
=================

The engine evaluates five independent institutional conditions.

Each condition contributes to a weighted confidence score.

Maximum Score:

```
1.00
```

Minimum Score:

```
0.00
```

=================================================================
STEP 1:
RANGE EFFICIENCY ANALYSIS
=========================

Purpose:

Determine whether volatility is compressing.

Calculation:

Recent Window:

```
Last 20 bars
```

Prior Window:

```
Previous 40 bars
```

Average Range:

```
High - Low
```

Logic:

```
recent_range < prior_range * 1.10
```

Interpretation:

If recent volatility is similar to or lower than prior volatility, price may be stabilizing.

Acceptance generally occurs when markets stop expanding and begin balancing.

Weight:

```
25%
```

=================================================================
STEP 2:
VALUE AREA STABILITY
====================

Purpose:

Measure auction balance.

Process:

1. Calculate Value Midpoint

   Average Close

2. Measure Distance Above Midpoint

3. Measure Distance Below Midpoint

4. Compare Symmetry

Formula:

balance_ratio =
smaller_deviation /
larger_deviation

Condition:

```
balance_ratio > 0.65
```

Interpretation:

Balanced auctions produce relatively equal excursions above and below perceived value.

A highly asymmetrical auction indicates directional dominance and weak acceptance.

Weight:

```
30%
```

Highest weighted factor in the model.

=================================================================
STEP 3:
VOLUME STABILITY ANALYSIS
=========================

Purpose:

Determine whether participation is stable.

Calculation:

Recent Average Volume

vs

Prior Average Volume

Condition:

Difference < 20%

Logic:

```
abs(recent_vol - prior_vol) /
prior_vol
```

Interpretation:

Acceptance generally occurs without dramatic participation shifts.

Volume spikes often signal:

• Discovery
• Breakouts
• Distribution
• Institutional repositioning

Stable volume suggests fair-value trading.

Weight:

```
20%
```

=================================================================
STEP 4:
TREND FLATNESS ANALYSIS
=======================

Purpose:

Ensure equilibrium is not mistaken for trend.

Calculation:

Linear Regression Slope

using:

```
np.polyfit()
```

against the last 20 closing prices.

The slope is compared against realized volatility.

Condition:

```
abs(slope) < volatility * 0.10
```

Interpretation:

True acceptance should exhibit minimal directional bias.

The engine avoids classifying slow trends as equilibrium.

Weight:

```
15%
```

=================================================================
STEP 5:
BREAKOUT FILTER
===============

Purpose:

Prevent breakout environments from being classified as acceptance.

Calculation:

Compare recent structure against prior structure.

Condition:

```
value_high <= prior_high
```

AND

```
value_low >= prior_low
```

Interpretation:

Price must remain contained inside the previously established range.

If price escapes the prior range:

```
Acceptance is invalidated.
```

This is one of the most important institutional safeguards in the model.

Weight:

```
10%
```

}
