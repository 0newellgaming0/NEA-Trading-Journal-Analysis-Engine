Wyckoff Fractal Lifecycle & Cross-Timeframe Structural Engine (v3.1)
1. Overview

This module implements a multi-timeframe structural market engine based on fractal lifecycle modeling, volume-price interaction, and regime classification inspired by the Wyckoff Method framework.

It transforms raw OHLCV data into:

Structural market phases (Accumulation → Absorption → Markup → Transition)
A fractal lifecycle state model
Cross-timeframe regime alignment (60M / Daily / Weekly)
An institutional strength score
Journal-ready structured output

The core objective is to approximate institutional behavior inference using statistical compression, slope dynamics, and volume-response asymmetry.

2. Design Philosophy

Markets are modeled as:

A liquidity-driven adaptive system transitioning through compression and expansion cycles

This engine assumes:

Price movement is not random; it is state-dependent
Volume is a proxy for institutional participation
Compression precedes expansion
Breakouts are valid only when supported by:
absorption
volatility contraction
structural pressure buildup

The model encodes these assumptions into continuous numeric state variables and then discretizes them into regime events.

STEP 3 — Lifecycle State Model

Function:

compute_lifecycle_state(df)

This is the core fractal intelligence engine.

It computes 4 key latent variables:

1. Accumulation Curve

Measures compression + trend stagnation:

High compression = potential accumulation zone
Penalized by strong slope (trend already established)
2. Absorption Saturation

Measures volume efficiency:

absorption=
price response
volume ratio
	​


Interpretation:

High volume + low price movement → absorption
3. Breakout Pressure

Combination of:

Accumulation
Absorption

Represents latent directional energy

4. Efficiency Decay

Measures:

Price movement per unit volume

High decay = inefficient movement (often institutional repositioning)

5. Liquidity Sweep Detection

Function:

detect_liquidity_sweep()

Detects engineered liquidity grabs:

Bearish Sweep (UTAD-like behavior)

Occurs when:

Price breaks above range high
Closes back inside range
Volume confirms displacement
Bullish Sweep (Spring-like behavior)

Occurs when:

Price breaks below range low
Recovers back into range
Volume confirms participation

Outputs:

{
  "type": "BullishSweep / BearishSweep",
  "confirmed": bool,
  "penetration_pct": float,
  "context": string
}

9. Breakout Validation Logic

Function:

is_valid_breakout()

A breakout is only valid if:

breakout_quality ≥ 0.6
absorption_depth ≥ 0.5
liquidity_displacement ≤ 0.7

This prevents false breakouts caused by:

low liquidity spikes
noise volatility
weak institutional participation
