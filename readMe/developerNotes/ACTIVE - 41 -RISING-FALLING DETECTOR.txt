Module Overview

The Rising / Falling Three Methods Module is a continuation-pattern strategy plugin designed for the NEA Event Architecture.

Unlike traditional candlestick scanners that only recognize completed patterns after the final breakout candle has formed, this module supports:

Historical Pattern Detection
Progressive Pattern Detection
Early Structure Recognition
Event Lifecycle Tracking
Trade Plan Generation
Resolver Integration
Journal Integration
Dashboard Integration

The module operates as a complete event-driven strategy component rather than a simple pattern detector.

Purpose

The objective of the module is to identify:

Bullish Continuation

Rising Three Methods

and

Bearish Continuation

Falling Three Methods

while also recognizing the structure before the final breakout candle appears.

This allows the framework to monitor developing continuation setups in real time rather than discovering them only after completion.

Why Progressive Detection Matters

Most candlestick systems behave like this:

Day 2
Day 3
Day 4
Day 5

Pattern Completed
Pattern Detected

This creates a lagging signal.

The NEA architecture improves this by detecting the pattern while it is forming.

Instead of waiting for Candle 5:

The module begins tracking the setup immediately.

When Candle 5 arrives:

Pattern Completed
Pattern Confirmed
Trade State Generated

This enables proactive monitoring rather than reactive discovery.

SEED State

Purpose:

Track incomplete structures.

Example:

Candle 1 Complete
Candle 2 Complete
Candle 3 Complete
Candle 4 Complete

Seed Event Created

The pattern is forming.

The breakout candle has not yet appeared.

CONFIRMED State

Occurs when:

Bullish

close > event["high"]

Bearish

close < event["low"]

Meaning:

The continuation breakout has occurred.

FAILED State

Occurs when:

Bullish

close < event["low"]

Bearish

close > event["high"]

Meaning:

The continuation thesis has broken.

EXPIRED State

Occurs when:

Maximum monitoring window exceeded

without confirmation.

Design Philosophy

The module treats Three Methods patterns as evolving market events.

Instead of asking:

Did the pattern occur?

it asks:

Is the pattern currently forming?

This aligns with institutional event tracking and transforms the pattern from a static historical observation into a monitored continuation opportunity.
