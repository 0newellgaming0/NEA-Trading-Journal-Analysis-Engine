📊 Harami Pattern Institutional Trading Engine
Overview

This module is a comprehensive institutional-grade candlestick analysis engine focused on detecting, validating, and generating trade plans around the Harami candlestick pattern within a broader market context.

It integrates:

Candlestick pattern recognition (Harami detection)
Volume-based institutional behavior analysis
Liquidity sweep detection (Wyckoff logic)
Trend + structure evaluation
Fibonacci alignment checks
Automated trade planning (entry/stop/targets)
Journal-ready formatted outputs

The system is designed for multi-factor confirmation trading, not standalone pattern signals.

🧠 Core Philosophy

This engine is built around the principle:

No pattern is valid without institutional confirmation.

A Harami pattern is only actionable when aligned with:

Trend structure (EMA / SMA alignment)
Volume confirmation (RVOL + spikes)
Liquidity behavior (sweeps / traps)
Market location (support / resistance)
Compression / expansion dynamics

🕯️ Harami Pattern Detection System
Core Detection
detect_harami_pattern(df)

Detects classic Harami structure:

Conditions:

Inside candle fully contained within mother candle body
Valid directional bias

Returns:

{
    detected: bool,
    type: BullishHarami | BearishHarami,
    high, low,
    body_high, body_low,
    inside_high, inside_low
}
Strength Classification
classify_harami_strength(c1, c2)

Measures:

body_ratio = inside_body / mother_body
Ratio	Strength
≤ 0.35	STRONG
≤ 0.75	WEAK
🚀 Expansion Candle Detection
detect_expansion_candle(df)

Identifies institutional momentum candles:

Conditions:

Volume spike (>2x previous)
Body expansion (>1.5x previous candle)

Output:

{
    detected: True,
    type: Bullish/Bearish,
    strength: Institutional Expansion
}
