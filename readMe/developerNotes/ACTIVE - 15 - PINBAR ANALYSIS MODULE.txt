# PINBAR MODULE

## Strategy Plugin (Resolver Compatible)

**Module Name:** `pinbar.py`
**Purpose:** Detect, track, validate, and manage Bullish and Bearish Pinbar reversal patterns through the Event Engine framework.

---

# Overview

The Pinbar Module is a fully self-contained candlestick strategy plugin designed for integration with the NEA event-driven architecture.

The module performs four primary responsibilities:

1. Detects bullish and bearish pinbar formations.
2. Creates structured event objects for the Event Engine.
3. Tracks confirmation and failure states over time.
4. Generates trade plans from validated pinbar setups.

Unlike traditional candlestick scanners that only identify patterns, this module treats each pinbar as an evolving market event that progresses through a lifecycle.

---

# Pinbar Theory

A pinbar is a rejection candle that demonstrates aggressive market rejection of a price area.

The candle consists of:

* A small real body
* One dominant wick
* A close near one end of the candle range

The long wick represents failed auction activity and liquidity rejection.

---

## Bullish Pinbar

A bullish pinbar indicates:

* Sellers pushed price lower
* Buyers absorbed supply
* Price closed near highs

Visual Example:

```text
    High
      |
      |
    [__]
      |
      |
      |
      |
      |
    Low
```

Characteristics:

* Long lower wick
* Small body
* Close near candle high

Interpretation:

Market rejected lower prices.

---

## Bearish Pinbar

A bearish pinbar indicates:

* Buyers pushed price higher
* Sellers absorbed demand
* Price closed near lows

Visual Example:

```text
      High
        |
        |
        |
        |
        |
      [__]
        |
       Low
```

Characteristics:

* Long upper wick
* Small body
* Close near candle low

Interpretation:

Market rejected higher prices.

# detect_pinbar()

## Purpose

Identifies whether a candle qualifies as a bullish or bearish pinbar.

---

## Signature

```python
detect_pinbar(candle, f)
```

---

## Parameters

### candle

Expected structure:

```python
{
    "Open": 100,
    "High": 105,
    "Low": 95,
    "Close": 104
}
```

### f

Safe float conversion function.

Example:

```python
float
```

or

```python
safe_float
```

---

## Detection Logic

### Range

```python
rng = high - low
```

---

### Body Size

```python
body = abs(close - open_)
```

---

### Wick Sizes

Upper Wick:

```python
upper = high - max(open_, close)
```

Lower Wick:

```python
lower = min(open_, close) - low
```

---

### Body Ratio

```python
body_ratio = body / rng
```

Measures how large the body is relative to total candle size.

Requirement:

```python
body_ratio <= 0.35
```

Body must be small.

---

### Wick Dominance Ratios

Bullish:

```python
bull_ratio = lower / body
```

Bearish:

```python
bear_ratio = upper / body
```

Requirement:

```python
>= 2.5
```

Dominant wick must be at least 2.5x body size.

---

### Close Position

```python
(close - low) / rng
```

Measures where the candle closed.

---

# Bullish Conditions

```python
bull_ratio >= 2.5
close_position >= 0.70
lower > upper
body_ratio <= 0.35
```

---

# Bearish Conditions

```python
bear_ratio >= 2.5
close_position <= 0.30
upper > lower
body_ratio <= 0.35
```

# build_pinbar_trade_state()

## Purpose

Converts a detected pinbar event into a complete trade plan.

---

## Signature

```python
build_pinbar_trade_state(event)
```

---

# Bullish Trade Logic

Entry:

```python
entry = high
```

Stop:

```python
low - (range * 0.1)
```

Target 1:

```python
high + range
```

Target 2:

```python
high + 2 * range
```

---

## Example

```text
High = 100
Low  = 90

Range = 10

Entry = 100
Stop  = 89

T1 = 110
T2 = 120
```

---

# Bearish Trade Logic

Entry:

```python
entry = low
```

Stop:

```python
high + (range * 0.1)
```

Target 1:

```python
low - range
```

Target 2:

```python
low - 2 * range
```

---

## Example

```text
High = 100
Low  = 90

Entry = 90
Stop  = 101

T1 = 80
T2 = 70
```

# Event Lifecycle

```text
PENDING
   |
   |
   +----> CONFIRMED
   |
   +----> FAILED
```

---

# Bullish Confirmation

Pattern confirms when:

```python
close > event["high"]
```

Meaning:

Price broke above pinbar high.

---

# Bullish Failure

Pattern fails when:

```python
close < event["low"]
```

Meaning:

Price violated rejection low.

---

# Bearish Confirmation

Pattern confirms when:

```python
close < event["low"]
```

Meaning:

Price broke below pinbar low.

---

# Bearish Failure

Pattern fails when:

```python
close > event["high"]
```

Meaning:

Price violated rejection high.

## Purpose

Scans market data, finds the latest pinbar, validates its lifecycle, and builds a trade plan.

---

## Signature

# Workflow

## Step 1

Search backwards through dataframe.

```python
for i in range(len(df)-1, -1, -1):
```

Newest patterns take priority.

---

## Step 2

Detect pinbar.

```python

```

---

## Step 3

Create event object.

Example:

```python
{
    "status": "PENDING",
    "direction": "Bullish",
    "high": 100,
    "low": 90
}
```

---

## Step 4

Validate future candles.

```python
pinbar_event_rules()
```

---

## Step 5

Update status.

Possible outcomes:

```text
PENDING
CONFIRMED
FAILED
EXPIRED
```

---

## Step 6

Generate trade plan.

```python
build_pinbar_trade_state()
```

# Current Limitations

The current implementation intentionally focuses on pure price-action detection and does not evaluate:

* Trend alignment
* Support/Resistance proximity
* Volume confirmation
* Institutional accumulation/distribution
* Wyckoff phase
* Liquidity sweep context
* Regime classification

As a result:

```python
"regime": "UNKNOWN"
```

is returned by default.

These contextual filters are expected to be added by higher-level framework modules.
