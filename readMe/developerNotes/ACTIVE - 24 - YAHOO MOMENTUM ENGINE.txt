Module: modules/yahoo_history.py
Function: load_yahoo_history()
1. Overview

yahoo_history.py is a refined institutional-grade historical data loader designed for multi-timeframe market analysis. It ingests locally stored OHLCV data (previously sourced from Yahoo Finance or equivalent pipelines) and produces:

Recent price/volume summary statistics
Momentum analysis using Awesome Oscillator (AO) and Accelerator Oscillator (AC)
Regime classification (trend state machine)
Multi-timeframe momentum alignment (15m / 60m / daily)

The module is optimized for trading dashboards, journal systems, and institutional-style market regime detection rather than raw data retrieval.

2. Core Purpose

This module is NOT a downloader. It is a:

Post-ingestion market intelligence engine

It assumes that historical CSV files already exist in:

modules/stock_data/<timeframe>/<TICKER>.csv

Its role is to transform that raw dataset into structured momentum and regime insights.

9. Momentum Engine (AO / AC)
9.1 Definition

The module implements a centralized:

Awesome Oscillator (AO)
Accelerator Oscillator (AC)
9.2 Formula Logic

Median price:

median = (high + low) / 2

AO:

AO = SMA(5, median) - SMA(34, median)

AC:

AC = AO - SMA(5, AO)
9.3 Interpretation Layers
AO State
Positive AO → short-term strength above equilibrium
Negative AO → short-term weakness below equilibrium
Zero → neutral balance
AO Trend
Rising AO → momentum expansion improving
Falling AO → momentum contraction increasing
Flat → equilibrium
AC State
Positive AC → acceleration building
Negative AC → deceleration present
Neutral → no acceleration bias
10. Regime Classification Engine

The module converts AO/AC into structured market regimes:

Bullish Expansion
AO > 0 and AC > 0

→ Trend strengthening

Bullish Pullback / Risk
AO > 0 and AC < 0

Sub-classified into:

Re-acceleration setup
Momentum weakening
Bearish Expansion
AO < 0 and AC < 0

→ Strong downtrend continuation

Bearish Exhaustion
AO < 0 and AC > 0

→ Potential reversal / recovery phase

Neutral / Transition

Fallback state when structure is unclear.

11. AO/AC Confirmation Signal

The system generates a binary confirmation:

ao_ac_confirm

Triggered when:

AO rising + AC positive
OR
AO falling + AC negative

This represents:

Momentum continuity confirmation
