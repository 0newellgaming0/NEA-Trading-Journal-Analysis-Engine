Event-driven structural detector of market turning points and break confirmations.

🔹 B) Detect Fractal Break Events

📍 5. FRACTAL PIVOT DETECTION
Function:
detect_fractal_pivot(df, i, window=2)
🔹 Purpose

Detects adaptive liquidity extremes using a dynamic window rather than a fixed 5-bar structure.

🔹 Logic

A candle is classified as a fractal pivot if:

🟢 Bullish Pivot (Liquidity Low)
Current low is the lowest within surrounding window
🔴 Bearish Pivot (Liquidity High)
Current high is the highest within surrounding window
🔹 Output Structure
{
  "type": "FRACTAL_PIVOT",
  "direction": "BullishPivot | BearishPivot",
  "high": float,
  "low": float,
  "index": int
}
🧠 Interpretation
Pivot Type	Meaning
Bullish Pivot	Liquidity sweep below structure
Bearish Pivot	Liquidity sweep above structure
⚡ 6. FRACTAL BREAK DETECTION
Function:
detect_fractal_break(event, high, low)
🔹 Purpose

Evaluates whether price has violated the fractal structure, triggering either:

breakout continuation
or structural failure
🔹 Rules
🟢 Bullish Pivot
BREAK_UP → high > fractal_high
FAIL → low < fractal_low
🔴 Bearish Pivot
BREAK_DOWN → low < fractal_low
FAIL → high > fractal_high
🧠 Key Principle

A fractal is only meaningful when it is broken or invalidated.

No break = no trade.
